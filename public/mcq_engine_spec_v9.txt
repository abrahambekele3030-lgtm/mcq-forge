2. SECOND SENTENCE: Explain WHY a SPECIFIC DISTRACTOR is plausible but wrong, identifying the SPECIFIC ERROR in reasoning or fact.

CONTENT RULES:

- MUST identify the specific reasoning error in the distractor.
- MUST NOT contain >30% of the words from the correct option.
- MUST provide ADDED VALUE beyond merely restating the question.

EXPLANATION VALIDATION CHECKLIST (MUST PASS ALL):
1. First sentence contains a NAMED CONCEPT from PDF structure.
2. First sentence DOES NOT contain the wording of the correct option.
3. Second sentence identifies a SPECIFIC DISTRACTOR (by letter or clear description).
4. Second sentence explains the SPECIFIC ERROR in that distractor.
5. Explanation provides educational insight beyond the question stem.
6. Word overlap with correct option is <=30%.

REJECTION RULES — REGENERATE ENTIRE QUESTION IF:
- Explanation contains >30% of the words from the correct option
- Explanation merely restates the question stem in different words
- Explanation fails to name a specific concept from PDF structure
- Explanation does not identify a specific distractor error
- Explanation uses circular reasoning ("X is true because X")

WORD OVERLAP CALCULATION:
- Tokenize both correct option and explanation
- Calculate: (shared words / total words in correct option) * 100%
- If >30%, reject

  Purpose: Formal one-sentence definition of the primary concept tested.

  Purpose: 2-3 sentence conceptual overview for context.

beginner | string
  Purpose: Lowest-level explanation using everyday analogies.
  Allowed: 20-30 words.

         Must use at least ONE concrete analogy.
  Validation: Word count 20-30.

intermediate | string
  Purpose: Standard explanation using PDF concept names.
  Allowed: 30-40 words.

         Must include the general pattern or rule that applies.
  Validation: Word count 30-40. Must contain at least one PDF concept name.

advanced | string
  Purpose: Deep explanation including mathematical or conceptual proof.
  Allowed: 40-50 words.
  Rules: Must reference the formula (if applicable) using LaTeX.

           For type E: minimum 3 steps. Each step is a complete sentence.

            "Step 2: Substitute values: \\(V = 3 \\times 4 = 12\\) V.",
            "Step 3: The answer is 12 V."]

reasoning_steps | array of strings
  Purpose: Logical steps to arrive at correct answer choice.
  Allowed: Minimum 2 steps for non-type-E. Each step is an imperative statement.
  Rules: Steps must be generalisable, not just "choose A".

            "Identify the number of products in the reaction.",
            "If reactants > products and products = 1, classify as combination."]

formula_analysis | array of objects
  Purpose: Breakdown of each formula used.
  Allowed: Empty array [] if no formula applies.

    variables:    array of objects {symbol: string (LaTeX), meaning: string, unit: string}
    derivation:   string or null — brief derivation if relevant

misconceptions | array of strings
  Purpose: Named specific misconceptions that lead students to wrong options.
  Allowed: Minimum 1 entry. Maximum 4 entries.
  Each entry: 15-40 words. Must describe a specific false belief and why it is wrong.
  Must NOT start with "A student thinks..." or any pedagogical framing.
  Valid form: "[False belief] is incorrect because [correct reasoning]."

related_concepts | array of strings
  Purpose: List of concepts that connect to the tested concept.
  Allowed: Minimum 2 entries. Each entry is a concept name string.

revision_note | string
  Purpose: Ultra-compact exam-ready cheat-sheet summary.
  Allowed: 8-20 words. Must be a standalone factual statement.
  Format: Concept: rule or formula. Example: "Combination: A + B -> AB. One product always."

glossary_entry | object
  Purpose: Structured glossary entry for glossary generation.
  Required fields:
    term:          string — exact concept name from PDF
    definition:    string — 20-40 word formal definition
    example_equation: string (LaTeX) or null — representative formula or null
    related_terms: array of strings — minimum 1 related term
  Validation: term must match a term present in the PDF. definition 20-40 words.

flashcard | object
  Purpose: Anki-compatible flashcard.
  Required fields:
    front: string — question form. Must end with "?". 5-20 words.
    back:  string — answer with supporting detail. 10-40 words.

  Validation: front ends with "?". back is >10 words.

concept_map_nodes | array of objects
  Purpose: Machine-readable concept map nodes for concept-map generation.
  Minimum: 2 nodes.
  Object structure:
    source:   string — source node label
    relation: string — edge label (e.g., "classified_as", "requires", "produces",
              "example_of", "opposite_of", "part_of", "causes", "measured_by")
    target:   string — target node label
  Validation: All node labels must be non-empty strings.
